# STAR-style README (copy into each project repo)

Use this under your project title on GitHub so your spoken answers match the site.

## Problem / Situation
- Who was the user, and what was broken or missing?

## Task
- What had to be true when you were done? (Performance, security, launch date, design fidelity.)

## Action (use “I”)
- What did **you** implement? Name the 2–4 highest-signal technical moves.
- If you chose A over B, add one line on **tradeoffs**.

## Result
- Prefer **numbers** you can stand behind (Lighthouse, latency, bundle size, data volume, conversion).
- If you do not have metrics, state a **qualitative outcome** you can defend in interview.

## Run locally
- Commands to install and start.

## Learned / Next
- One sentence on what you would extend next (shows growth without selling short).
